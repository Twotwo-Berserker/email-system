## 02 实训内容介绍

### 邮件系统功能需求

1. **邮箱注册**
邮件系统提供注册功能，用户可注册邮箱，服务器将注册信息存入数据库，用于登录身份验证。

2. **接收邮件**
用户登录邮箱后，程序自动连接邮件服务器下载新邮件；可使用 IMAP/POP3 协议实现，下载完成后在收件箱展示邮件。

3. **发送邮件**
用户可通过客户端发送邮件，基于 SMTP 协议与服务器通信；需支持填写收件人、抄送、主题、邮件正文、上传附件。

4. **其他基础功能**
标记邮件已读 / 未读、删除邮件、邮件搜索、邮件同步等。

### 智能系统 \- 1（可插拔插件融合至邮件服务器）

1. 基于机器学习 / 深度学习的垃圾邮件识别插件

2. 邮件优先级排序、预测推送：基于用户历史打开、回复行为预测邮件重要性，自动置顶高优先级邮件、推送通知

3. 安全与多媒体处理：AI 识别伪造发件人、恶意链接等复杂安全威胁

### 智能系统 \- 2 插件加载方案

1. 传统机器学习插件：以 DLL 动态链接库形式提供，服务启动时动态加载

2. 大模型深度学习插件：调用第三方大模型 SDK，用户自主填写 API 密钥开启功能

3. 任务异步 / 实时区分：

    - 实时任务：垃圾邮件过滤（邮件到达立即分析）

    - 异步任务：邮件摘要生成，避免阻塞系统主线程

### 传输协议实现方案

1. **标准协议实现**：基于原生 SMTP、POP3 协议开发

2. **协议扩展**：底层沿用 SMTP/POP3，上层自定义消息格式、权限、加密规则

3. **自定义私有协议**：基于 TCP、WebSocket、HTTP 二次封装自定义通信协议

### 核心协议说明

#### SMTP 协议

SMTP（Simple Mail Transfer Protocol，简单邮件传输协议），标准邮件发送协议。

1. 作用：将邮件从客户端发送至收件人邮件服务器

2. 特性：支持身份验证、加密，保障邮件隐私与完整性；支持纯文本、HTML、附件多媒体邮件

3. 局限：仅负责发送邮件，接收邮件需搭配 POP3/IMAP 协议

4. 参考资料

    - [https://zhuanlan\.zhihu\.com/p/522806299](https://zhuanlan.zhihu.com/p/522806299)

    - [https://www\.rfc\-editor\.org/info/rfc788](https://www.rfc-editor.org/info/rfc788)

5. SMTP 核心示例代码

```c++
int sockfd = socket(AF_INET, SOCK_STREAM, 0);
ret = connect(sockfd, &server_addr, sizeof(server_addr));
n = recv(sockfd, buff, BUFF_SIZE, 0);
std::string ehlo_msg = std::format("EHLO {}\r\n", "163.com");
n = send(sockfd, ehlo_msg.c_str(), ehlo_msg.length(), 0);
n = recv(sockfd, buff, BUFF_SIZE, 0);
std::string auth_msg = std::string("AUTH LOGIN\r\n");
n = send(sockfd, auth_msg.c_str(), auth_msg.length(), 0);
n = recv(sockfd, buff, BUFF_SIZE, 0);
std::string mail_base64 = base64_encode(std::string("123456789@163.com"));
std::string mail_s = mail_base64 + "/r/n";
n = send(sockfd, mail_s.c_str(), mail_s.length(), 0);
n = recv(sockfd, buff, BUFF_SIZE, 0);
std::string auto_base64 = base64_encode(std::string("auto code"));
std::string auth_s = auto_base64 + "/r/n";
n = send(sockfd, auth_s.c_str(), auth_s.length(), 0);
n = recv(sockfd, buff, BUFF_SIZE, 0);
```

#### POP3 协议

POP3（Post Office Protocol Version 3，邮局协议第三版），标准邮件接收协议。

1. 特性：单向下载邮件至本地，默认不保留服务器副本；仅单设备使用，多端无法同步邮件状态

2. 优点：邮件下载至本地后可断开网络，节省流量、设备功耗

3. 缺陷：不支持多端同步已读、删除状态，多设备使用需手动同步

4. 参考资料：[https://www\.rfc\-editor\.org/info/rfc2449](https://www.rfc-editor.org/info/rfc2449)

### 自定义报文约定

1. HTTP 层：使用 GET/POST/DELETE 等方法完成数据库增删改查

2. TCP 底层：自定义报文结构，内部分支处理不同业务

3. 自定义报文字段：功能码、发送方地址、接收者地址、主题、正文长度、正文、附件信息、时间戳

### 项目实现细节

1. **Web 客户端**：前后端分离

    - 后端框架不限：SpringBoot \(Java\)、Django \(Python\)、Beego \(Golang\) 等

    - 前端推荐 Vue 开发

2. **桌面 APP 客户端**：QT 开发界面

3. 通信可选方案：不强制原生 SMTP/POP3，可基于 HTTP 自定义上层消息封装协议

4. 数据库设计：绘制 E\-R 图，分多表存储数据；邮件正文存入数据库，附件单独存磁盘文件做分离存储

5. 前端 UI：参考主流成熟邮件系统布局交互

### 项目加分项

- 自定义通信报文设计合理规范

- 服务端数据库分层、分表设计合理

- 客户端与服务器双向邮件状态同步

- 实现全局邮件搜索、自定义邮件过滤

- 新邮件实时推送通知

- 简洁友好、交互完善的前端 UI 界面

- 客户端本地邮件缓存功能
